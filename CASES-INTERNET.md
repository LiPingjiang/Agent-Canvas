# Agent Canvas — 互联网行业 Case 专项深挖

> 对互联网行业的每个部门、每个岗位做逐点排查，找出可视化与信息表达的覆盖盲区。
> 在现有 205 个 case 基础上，新增 75 个互联网行业专属 case。

---

## 一、互联网行业组织架构全景

互联网公司比传统企业的部门更细分、角色更专业化。典型架构：

```
管理层：CEO / CTO / CPO / CMO / COO
│
├── 产品部 — PM、PO、产品运营
├── 设计部 — UX设计师、UI设计师、交互设计师、视觉设计师
├── 前端开发部 — Web前端、小程序、H5
├── 移动端开发部 — iOS、Android、Flutter/RN
├── 后端开发部 — 服务端、中间件、基础架构
├── 算法/AI部 — 推荐算法、搜索、NLP、CV、大模型
├── 数据部 — 数据工程、数据分析、数据科学、BI
├── 测试部(QA) — 功能测试、自动化测试、性能测试
├── 运维/SRE部 — SRE、DevOps、基础架构
├── 安全部 — 安全开发、安全运营(SOC)、渗透测试
├── 增长团队 — 用户增长、商业化增长
├── 运营部 — 用户运营、内容运营、活动运营
├── 市场部 — 品牌、效果投放、PR
├── 商业化部 — 广告、会员、交易
├── 国际化部 — 出海、本地化
├── 职能部门 — HR、财务、法务、行政（已在通用部门覆盖）
```

### 3.5 关键发现

**互联网行业是可视化需求最密集的领域**。75 个新增 case 中，算法/AI(9个)和 SRE(8个)是最丰富的场景来源——这两个岗位几乎每天都在与数据可视化和信息表达打交道。

**算法/AI 部门是此前最大的盲区**。模型训练曲线、混淆矩阵、SHAP 图、推荐漏斗、A/B 实验报告——这些都是算法工程师每天必看的可视化，但此前完全缺失。这组 case 对组件库提出了新的要求：需要支持科学可视化（训练曲线、PR/ROC）和统计可视化（混淆矩阵、置信区间）。

**SRE/DevOps 的可视化有独特模式**。SLO 燃尽率/错误预算是一种独特的可视化形态——它不是简单的趋势线，而是"预算消耗速度 vs 剩余时间"的动态跟踪。Kill Chain 攻击链是安全领域特有的流程图变体。DAG 依赖图需要自动布局算法。

**增长团队的可视化以"对比+实验"为核心**。同期群留存热力图、LTV/CAC 散点、A/B 实验矩阵——这些都不是传统图表，而是增长分析师特有的"实验驱动"可视化模式。

**商业化/变现是全新的场景类别**。广告填充率、eCPM 趋势、会员转化漏斗——这些场景需要支持广告技术领域的特定指标和展示方式。
