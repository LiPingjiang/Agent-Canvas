# Agent Canvas

> Agent AGI 可视化生成工具 — 让 Agent 用语义描述而非代码来生成专业级可视化页面。

## 为什么需要 Agent Canvas？

当 Agent 需要生成可视化时，现有方案都有根本性限制：

- **Markdown 渲染** — 最可靠，但表达力极弱，只能做简单排版
- **HTML/CSS/JS 生成**（如 Claude Artifacts）— 灵活但不可靠，代码可能出错，审美不稳定
- **Python 绘图** — 灵活但可靠性差，审美不可控，需要运行时环境
- **DSL 配置**（如 ECharts/Vega-Lite）— 可靠但受限于特定图表类型

Agent Canvas 的核心思路是**分离关注点**：让 Agent 专注于"展示什么"（生成语义化 JSON Schema），让确定性渲染引擎处理"怎么渲染"（专业级组件 + 自动布局），并内置视觉反馈闭环（渲染 → 截图 → 多模态审查 → 修正）。

## 核心设计

三大设计原则：

**语义驱动** — Agent 生成结构化 JSON Schema 描述视觉意图，而非手写前端代码。

**有界生成 + 确定性渲染** — Agent 只能使用预定义组件，渲染引擎保证视觉质量和一致性。

**视觉反馈闭环** — 渲染 → 截图 → 多模态 LLM 审查 → 反馈修正，自动保障输出质量。

## 认知框架：11 种可视化原子模式

任何可视化都是以下模式的组合：

**描述性模式**（描述事物状态）：

| 模式 | 核心问题 | 典型形态 |
|------|---------|---------|
| Compare（比较） | A 和 B 有什么不同？ | 柱状图、雷达图、对比卡片 |
| Compose（构成） | 整体由哪些部分构成？ | 饼图、堆叠柱、树形图 |
| Connect（关联） | 谁和谁有关系？ | 网络图、关系图、ER 图 |
| Flow（流转） | 东西从哪来到哪去？ | 流程图、桑基图、序列图 |
| Arrange（组织） | 东西怎么组织的？ | 组织架构图、思维导图、看板 |
| Track（追踪） | 随时间怎么变化？ | 折线图、甘特图、时间线 |
| Distribute（分布） | 数据怎么分布的？ | 直方图、散点图、热力图 |
| Narrate（叙事） | 故事脉络是什么？ | 报告布局、信息图、演示文稿 |

**操作性模式**（支持人类行动）：

| 模式 | 核心问题 | 典型形态 |
|------|---------|---------|
| Decide（决策） | 应该选哪个？ | 决策树、评分矩阵、利弊对比 |
| Simulate（模拟） | 如果...会怎样？ | 预测曲线、敏感性龙卷风图 |
| Navigate（导航） | 怎么从这里到那里？ | 地图导航、路径规划 |

## Case 覆盖度

累计 **301 个 case**，覆盖：

- **19/20** 个国民经济行业门类（GB/T 4754-2017）
- **12** 个通用企业部门 + **15** 个互联网部门 + **20** 个行业特有部门
- **15+** 类岗位（从 CEO 到算法工程师、SRE、增长黑客）
- **30+** 种可视化形态（从仪表盘/报告到混淆矩阵/K线图/技术雷达）
- **10** 种企业类型（大企业/创业公司/公共机构/非营利组织/医院/学校等）

## 技术栈

| 层 | 技术 |
|----|------|
| 渲染引擎 | React 18+ / TypeScript |
| 组件样式 | Tailwind CSS / CSS Variables |
| 图表渲染 | Recharts / ECharts |
| Schema 验证 | Zod |
| 截图服务 | Playwright (headless) |
| 视觉审查 | 多模态 LLM (GPT-4o / Claude 3.5 Sonnet) |
| 构建工具 | Vite |

## 实施路线图

- **Phase 1** — 核心框架 + 基础组件（Canvas DSL / 渲染引擎 / 8 个核心组件）
- **Phase 2** — 组件完善 + 模板系统（图表/图/布局全覆盖 + 预设模板）
- **Phase 3** — 视觉反馈闭环（截图服务 / 多模态审查 / 自动修正）
- **Phase 4** — Agent 集成 + 优化（提示词模板 / Schema 约束 / 性能优化）
- **Phase 5** — 高级特性（动画 / 交互 / 实时数据 / 多页面 / 导出）
